# Документирование в IT-проектах
## Цель: научится формировать необходимую документацию для разработки нового продукта.

### Контекст 
Необходимо подготовить документацию для проекта по реализации приложения «Запись на прием» для медицинской клиники.

### Задача: 
1. Создать Customer journey map (CJM) от лица клиента медицинской клиники;
2. На основании CJM подготовить техническое задание;
3. Создать план-график работ, включающий запуск продукта с минимальным функционалом (MVP) и егопоследующие доработки.

### Решение
Первоочередно выделил основные возможные этапы взаимодействия продукта с пользователем, определил точки контакта на каждом этапе, сформировал действия пользователя. Объединив все в одном месте, сформировал полноценный [CJM](https://github.com/Aptyshev97/Portfolio/blob/main/%D0%94%D0%BE%D0%BA%D1%83%D0%BC%D0%B5%D0%BD%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5%20%D0%B2%20%D0%BF%D1%80%D0%BE%D0%B5%D0%BA%D1%82%D0%B0%D1%85/CJM%20%D0%97%D0%B0%D0%BF%D0%B8%D1%81%D1%8C%20%D0%BD%D0%B0%20%D0%BF%D1%80%D0%B8%D0%B5%D0%BC.pdf). Далее, основываясь на CJM составил [техническое задание](https://github.com/Aptyshev97/Portfolio/blob/main/%D0%94%D0%BE%D0%BA%D1%83%D0%BC%D0%B5%D0%BD%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5%20%D0%B2%20%D0%BF%D1%80%D0%BE%D0%B5%D0%BA%D1%82%D0%B0%D1%85/%D0%A2%D0%97%20%D0%93%D0%9E%D0%A1%D0%A2%2019%20%D0%97%D0%B0%D0%BF%D0%B8%D1%81%D1%8C%20%D0%BD%D0%B0%20%D0%BF%D1%80%D0%B8%D0%B5%D0%BC.pdf) по ГОСТ 19.  На основании технического задания сформировал [план-график](https://docs.google.com/spreadsheets/d/18ZoKIL5nbq9VVmv8NZQrvwaT59GehYx2duSPiHFDZic/edit#gid=0) , в котором отобразил backlog и спринты с датами.
