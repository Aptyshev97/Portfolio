# Документирование в IT-проектах
## Цель: научится формировать необходимую документацию для разработки нового продукта.

### Контекст 
Вам необходимо подготовить документацию для проекта по реализации приложения «Запись на прием» для медицинской клиники.

### Задача: 
1. Создать Customer journey map (CJM) от лица клиента медицинской клиники;
2. На основании CJM подготовить техническое задание;
3. Создать план-график работ, включающий запуск продукта с минимальным функционалом (MVP) и егопоследующие доработки.

### Решение
Первоочередно выделил основные возможные этапы взаимодействия продукта с пользователем, определил точки контакта на каждом этапе, сформировал действия пользователя. Объединив все в одном месте, сформировал полноценный [CJM](). Далее, основываясь на CJM составил [техническое задание]() по ГОСТ 19.  На основании технического задания сформировал [план-график](https://docs.google.com/spreadsheets/d/18ZoKIL5nbq9VVmv8NZQrvwaT59GehYx2duSPiHFDZic/edit#gid=0) , в котором отобразил backlog и спринты с датами.
